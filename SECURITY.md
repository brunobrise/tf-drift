# Security Policy

## Supported Versions

We actively support and provide security updates for the following versions of `tf-drift`:

| Version | Supported |
| :--- | :--- |
| v1.x | :white_check_mark: Yes |
| < v1.x | :x: No |

---

## Reporting a Vulnerability

We take the security of our project seriously. If you believe you have found a security vulnerability in `tf-drift`, please do **not** report it via a public GitHub issue. Instead, disclose it responsibly by following these steps:

1. Send an email to **security@brunobrise.com** with details of the vulnerability.
2. Include a description of the issue, steps to reproduce, and any potential impact.
3. We will acknowledge receipt of your vulnerability report within 48 hours and work with you to resolve it.

Once a fix is ready, we will publish a new release and document the vulnerability in our release notes. Thank you for helping keep this project secure!
